핵심 API 설명

1 GET /subjects

기능
등록된 모든 과목 목록을 조회함

사용 목적
수행평가를 등록하기 전에 어떤 과목이 있는지 확인하기 위해 사용함

처리 흐름
FastAPI 서버가 요청을 받음
service 계층의 get_subjects 함수가 실행됨
데이터베이스에서 Subject 테이블의 모든 데이터를 조회함
조회된 과목 목록을 응답으로 반환함

응답 내용
과목 id
과목명
과목 분류
이수 단위

2 POST /subjects

기능
새로운 과목을 등록함

사용 목적
수행평가를 과목별로 관리하기 위해 먼저 과목 정보를 저장함

요청 내용
name 과목명
category 과목 분류
credit_hours 이수 단위

처리 흐름
사용자가 과목 정보를 입력함
FastAPI 서버가 요청 데이터를 Pydantic 스키마로 검증함
service 계층의 create_subject 함수가 실행됨
Subject 테이블에 새 과목을 저장함
저장된 과목 정보를 응답으로 반환함

3 GET /assessments

기능
등록된 모든 수행평가 목록을 조회함

사용 목적
현재 등록된 수행평가 전체 일정을 확인하기 위해 사용함

처리 흐름
FastAPI 서버가 요청을 받음
service 계층의 get_assessments 함수가 실행됨
데이터베이스에서 PerformanceAssessment 테이블의 모든 데이터를 조회함
조회된 수행평가 목록을 응답으로 반환함

응답 내용
수행평가 id
과목 id
수행평가 유형
반영 비율
마감일
겹치는 과제 수
실제 소요 시간
스트레스 점수

4 POST /assessments

기능
새로운 수행평가를 등록함

사용 목적
과목별 수행평가 정보를 저장하고 이후 분석 기능에 활용하기 위해 사용함

요청 내용
subject_id 과목 id
evaluation_type 수행평가 유형
weight 반영 비율
due_date 마감일
overlapping_tasks 겹치는 과제 수

처리 흐름
사용자가 수행평가 정보를 입력함
FastAPI 서버가 요청 데이터를 Pydantic 스키마로 검증함
입력된 subject_id에 해당하는 과목이 존재하는지 확인함
과목이 없으면 404 오류를 반환함
과목이 있으면 service 계층의 create_assessment 함수가 실행됨
PerformanceAssessment 테이블에 새 수행평가를 저장함
저장된 수행평가 정보를 응답으로 반환함

5 POST /assessments/{assessment_id}/feedback

기능
수행평가 완료 후 실제 소요 시간과 스트레스 점수를 기록함

사용 목적
사용자의 실제 경험 데이터를 저장하여 과목별 스트레스 분석에 활용함

요청 내용
actual_time_spent 실제 소요 시간
user_stress_score 사용자가 느낀 스트레스 점수

처리 흐름
사용자가 수행평가 id와 피드백 정보를 입력함
FastAPI 서버가 요청 데이터를 검증함
service 계층의 update_feedback 함수가 실행됨
해당 id의 수행평가를 데이터베이스에서 찾음
수행평가가 없으면 404 오류를 반환함
수행평가가 있으면 실제 소요 시간과 스트레스 점수를 수정함
수정된 수행평가 정보를 응답으로 반환함

6 GET /dashboard/summary

기능
수행평가 폭탄주의보 대시보드 정보를 조회함

사용 목적
현재 날짜부터 7일 이내에 마감되는 수행평가가 몇 개인지 확인하고 위험도를 판단하기 위해 사용함

처리 흐름
FastAPI 서버가 요청을 받음
service 계층의 get_dashboard_summary 함수가 실행됨
현재 날짜와 7일 뒤 날짜를 계산함
PerformanceAssessment 테이블에서 마감일이 해당 기간 안에 있는 수행평가를 조회함
조회된 수행평가 개수를 계산함
개수가 4개 이상이면 위험으로 판단함
개수가 2개 이상이면 경고로 판단함
그 외에는 보통으로 판단함
상태와 안내 문구를 응답으로 반환함

응답 내용
마감임박 수행평가 개수
상태
안내 문구

기술 요소
datetime과 timedelta를 활용한 날짜 계산
SQLAlchemy filter와 between을 활용한 기간 조건 조회
if elif else를 활용한 상태 판별
리스트 길이 계산을 통한 데이터 집계

7 GET /assessments/priority

기능
수행평가 우선순위 목록을 조회함

사용 목적
마감일이 가깝고 반영 비율이 높은 수행평가를 먼저 확인하기 위해 사용함

처리 흐름
FastAPI 서버가 요청을 받음
service 계층의 get_assessment_priority 함수가 실행됨
데이터베이스에서 모든 수행평가를 조회함
각 수행평가마다 마감일까지 남은 일수를 계산함
이미 지난 수행평가는 제외함
반영 비율과 남은 일수를 조합해 우선순위 점수를 계산함
우선순위 점수가 높은 순서대로 정렬함
정렬된 수행평가 목록을 응답으로 반환함

응답 내용
수행평가 id
과목명
마감일
반영 비율
남은 일수
우선순위 점수

기술 요소
for 반복문을 활용한 수행평가 순회
날짜 차이 계산을 통한 남은 일수 계산
가중치 계산 알고리즘 적용
sorted 함수를 활용한 정렬
ORM relationship을 활용한 과목명 조회

8 GET /analytics/user-report

기능
과목별 스트레스 분석 리포트를 조회함

사용 목적
사용자가 어떤 과목에서 스트레스를 많이 받는지 확인하기 위해 사용함

처리 흐름
FastAPI 서버가 요청을 받음
service 계층의 get_user_report 함수가 실행됨
데이터베이스에서 모든 과목을 조회함
각 과목에 연결된 수행평가 목록을 확인함
스트레스 점수가 기록된 수행평가만 모음
과목별 평균 스트레스 점수를 계산함
과목별 분석 결과와 요약 문구를 응답으로 반환함

응답 내용
과목명
평균 스트레스
평가 개수
요약 문구

기술 요소
SQLAlchemy relationship을 활용한 과목과 수행평가 연결
for 반복문을 활용한 과목별 데이터 집계
sum과 len을 활용한 평균 계산
조건문을 활용한 빈 데이터 제외

9 API 설계 특징

main py는 API 엔드포인트를 담당함
service py는 실제 분석 로직과 데이터 처리 로직을 담당함
schemas py는 요청과 응답 데이터 형식을 검증함
models py는 데이터베이스 테이블 구조를 정의함
database py는 데이터베이스 연결을 담당함

API와 로직을 분리하여 코드 유지보수가 쉬워짐
과목 테이블과 수행평가 테이블을 ORM 관계로 연결하여 2개 이상의 테이블 관계 조건을 만족함
대시보드 통계, 우선순위 추천, 스트레스 리포트를 통해 수행평가 관리 문제를 해결함
